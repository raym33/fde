# Service blueprint: AI governance rollout

Objective: establish a lightweight but operational AI governance model that prioritizes use cases, controls risk, and measures ROI.

Required outputs:
- governance operating model
- initial AI use case portfolio
- approval workflow for new initiatives
- risk register structure
- KPI and audit plan

Implementation requirements:
- Inventory existing AI usage before proposing net-new tools.
- Classify initiatives by business criticality, sensitivity, regulatory risk, and owner.
- Define when local, hybrid, or frontier models are allowed.
- Require human sign-off for policy, legal, finance, and safety-sensitive outputs.
- Include a compact review cadence for leadership.

Technical checklist:
1. Create a baseline portfolio of current and proposed AI use cases.
2. Define minimum controls: owner, approved data sources, output review, logging, rollback.
3. Define escalation path for high-risk tasks and external model usage.
4. Define ROI tracking: hours saved, errors reduced, cost per workflow, adoption rate.
5. Define policy update cycle and operator responsibilities.

Expected delivery artifact:
- governance charter
- use case scoring model
- control matrix
- 90-day rollout plan
- audit checklist
- executive review pack outline
